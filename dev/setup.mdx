## Installation

<Steps>
  <Step title="Clone the Repository">
    Clone the EducaStream repository to your local machine:
    
    ```bash
    git clone <repository-url>
    cd educastream
    ```
  </Step>

  <Step title="Install Dependencies">
    Install all required packages using your preferred package manager:
    
    <CodeGroup>
    ```bash npm
    npm install
    ```

    ```bash yarn
    yarn install
    ```

    ```bash pnpm
    pnpm install
    ```
    </CodeGroup>

    This will install all dependencies from `package.json`, including:
    - React 18.2.0
    - Firebase 10.5.2
    - Stripe 14.3.0
    - React Router 6.17.0
    - And more...
  </Step>

  <Step title="Environment Configuration">
    Create a `.env` file in the root directory and configure your environment variables:
    
    ```bash
    touch .env
    ```

    Add the following variables (see [Firebase Config](/dev/firebase-config) and [Stripe Integration](/dev/stripe-integration) for values):

    ```bash
    # Firebase Configuration
    VITE_API_KEY=your_firebase_api_key
    VITE_AUTH_DOMAIN=your_project.firebaseapp.com
    VITE_PROJECT_ID=your_project_id
    VITE_STORAGE_BUCKET=your_project.appspot.com
    VITE_MESSAGING_SENDER_ID=your_sender_id
    VITE_APP_ID=your_app_id

    # Stripe Configuration (if needed for frontend)
    VITE_STRIPE_PUBLIC_KEY=your_stripe_public_key
    ```

    <Warning>
      Never commit your `.env` file to version control. Add it to `.gitignore`.
    </Warning>
  </Step>

  <Step title="Start Development Server">
    Launch the Vite development server:
    
    <CodeGroup>
    ```bash npm
    npm run dev
    ```

    ```bash yarn
    yarn dev
    ```

    ```bash pnpm
    pnpm dev
    ```
    </CodeGroup>

    The application will start at `http://localhost:5173` (default Vite port).
  </Step>
</Steps>

## Backend Configuration

The frontend connects to a Railway-hosted backend API. The base URL is configured in `main.jsx:8`:

```javascript
import axios from "axios";

axios.defaults.baseURL = "https://backend-educastream-production.up.railway.app/";
```

<Info>
  If you're running a local backend, update this URL to `http://localhost:YOUR_PORT`
</Info>

## Environment Variables

EducaStream uses Vite's environment variable system. Variables must be prefixed with `VITE_` to be exposed to the client:

<CodeGroup>
```javascript Firebase Config
const firebaseConfig = {
  apiKey: import.meta.env.VITE_API_KEY,
  authDomain: import.meta.env.VITE_AUTH_DOMAIN,
  projectId: import.meta.env.VITE_PROJECT_ID,
  storageBucket: import.meta.env.VITE_STORAGE_BUCKET,
  messagingSenderId: import.meta.env.VITE_MESSAGING_SENDER_ID,
  appId: import.meta.env.VITE_APP_ID,
};
```

```bash .env File
VITE_API_KEY=AIzaSy...
VITE_AUTH_DOMAIN=your-app.firebaseapp.com
VITE_PROJECT_ID=your-project-id
VITE_STORAGE_BUCKET=your-app.appspot.com
VITE_MESSAGING_SENDER_ID=123456789
VITE_APP_ID=1:123456789:web:abc123
```
</CodeGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Port already in use">
    If port 5173 is already in use, Vite will automatically try the next available port. You can also specify a custom port:
    
    ```javascript vite.config.js
    export default defineConfig({
      plugins: [react()],
      server: {
        port: 3000
      }
    });
    ```
  </Accordion>

  <Accordion title="Module not found errors">
    Delete `node_modules` and reinstall:
    
    ```bash
    rm -rf node_modules package-lock.json
    npm install
    ```
  </Accordion>

  <Accordion title="Firebase initialization fails">
    Ensure all Firebase environment variables are set correctly in `.env`:
    
    - Check for typos in variable names
    - Verify all values from Firebase Console
    - Restart dev server after changing `.env`
  </Accordion>

  <Accordion title="Axios network errors">
    If API calls fail:
    
    1. Check backend is running
    2. Verify the `axios.defaults.baseURL` in `main.jsx`
    3. Check CORS settings on backend
    4. Inspect network tab in browser DevTools
  </Accordion>
</AccordionGroup>
